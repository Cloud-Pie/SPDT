Always Resize Strategy

Description

In this strategy, reconfigurations using vertical scaling at the level of containers as well as at the level of virtual machines is explored.
This strategy not only adds extra capacity by adding new VMs but it may also replace VMs with a more cost-efficient type. However, to migrate or replace a VM type an overlapping between the initial VM set S1 and the desired VM set S2 is required.
It means that during the transition of the two states both configurations would add an overhead on costs in order to meet the QoS.
